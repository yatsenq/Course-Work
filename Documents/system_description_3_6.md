# 3.6. Опис функціоналу розробленої системи

Розроблена інтелектуальна система аналізу новин являє собою веб-застосунок на базі фреймворку Flask, що інтегрує сучасні моделі машинного навчання для детекції фейків та класифікації тематики україномовного контенту.

## 1. Огляд інтерфейсу та сторінок застосунку

### Головна сторінка (index.html)
Виконує роль лендінгу та інформаційного хабу. На ній відображаються:
*   **Глобальна статистика:** Інтерактивна діаграма (Chart.js), що показує розподіл перевірених новин за темами.
*   **Лічильники реального часу:** Загальна кількість перевірок та кількість виявлених фейків за останню добу.
*   **Опис можливостей:** Картки з ключовими перевагами системи (детекція, тематика, PDF-звіти).

### Сторінка аналізу (dashboard.html)
Основний робочий простір користувача, що містить:
*   **Форму введення:** Текстове поле для вставки новини (підтримує до 3000 символів).
*   **Інструменти швидкого заповнення:** Кнопки для автоматичного завантаження прикладів новин з реальних джерел (Укрінформ) або прикладів маніпуляцій (StopFake) через API-запити.
*   **Блок результатів:** Після аналізу динамічно з'являється панель з вердиктом (True/Fake), відсотком впевненості моделі, визначеною категорією та виділеними маркерами маніпуляцій у тексті.

### Сторінка історії (history.html)
Доступна авторизованим користувачам для перегляду попередніх перевірок.
*   **Таблиця записів:** Містить дату перевірки, початковий текст (уривок), вердикт моделі та тему.
*   **Управління:** Можливість завантажити детальний PDF-звіт для кожного запису або очистити історію.

### Сторінки автентифікації (login.html, register.html)
Реалізовані з дотриманням стандартів безпеки:
*   Форми мають валідацію полів на стороні клієнта та сервера.
*   Використовується Flask-Flash для сповіщень про успішний вхід або помилки.

## 2. Функціональні можливості користувачів

*   **Авторизований користувач:** Має повний доступ до аналізу, ведення персональної історії перевірок, отримання PDF-звітів та можливість залишати фідбек до результатів роботи моделей.
*   **Гість (Guest):** Може виконувати анонімний аналіз новин без збереження історії в базі даних.
*   **Генерація PDF-звітів:** Система використовує бібліотеку `ReportLab` для створення професійних звітів, що містять повний текст новини, детальні показники всіх моделей та дату аналізу.

## 3. Маршрути Flask (API та Навігація)

Система реалізує наступні ключові маршрути:
*   `GET /`: Головна сторінка, збір статистики з БД.
*   `GET /dashboard`: Сторінка аналізу (доступна для всіх).
*   `POST /dashboard`: Обробка тексту, виклик ML-сервісу, збереження результату в БД.
*   `GET /api/random_example/<type>`: Повертає випадкову новину (True/Fake) шляхом парсингу зовнішніх ресурсів (BeautifulSoup4).
*   `GET /history`: Відображення списку перевірок поточного користувача.
*   `GET /download-report`: Генерація та видача PDF-файлу.
*   `POST /admin/kick/<user_id>`: (Адмін) Активація/деактивація облікових записів.

## 4. Сервіс машинного навчання (ml_service.py)

Сервіс реалізує складний конвеєр обробки даних:
1.  **Завантаження моделей:** Використовується `huggingface_hub` для динамічного підтягування ваг BERT та логістичної регресії. Система автоматично адаптується під різні архітектури класифікаційних голів (Head A, B, C).
2.  **Інференс:** 
    *   Текст проходить токенізацію (`BertTokenizerFast`).
    *   Паралельно працюють 4 моделі (ансамбль): класичні (LogReg, SVM) та глибокі нейронні мережі (BERT).
    *   Виконується морфологічний аналіз для пошуку маркерів маніпуляцій.
3.  **Вихідні дані:** Сервіс повертає JSON-об'єкт з усередненим вердиктом, впевненістю кожної моделі окремо та масивом знайдених ключових слів.

## 5. Безпека та автентифікація

*   **Паролі:** Зберігаються виключно у вигляді хешів (алгоритм `scrypt`), що унеможливлює їх викрадення в чистому вигляді.
*   **Керування доступом:** Використовується `Flask-Login` з перевіркою прапорця `is_active`.
*   **Захист сесій:** Реалізовано через статичний `SECRET_KEY` та захищені Cookie, що забезпечує стабільну роботу в хмарних середовищах.
